Host & container packet policy

3 independent layers · fail-closed
firewalld
host-terminated input · source-scoped rich rules · zone ownership by live UUID. Not trusted to protect Docker-published ports alone.
iptables DOCKER-USER
atomic chain installed before Docker starts; reasserted by a firewalld D-Bus reload watcher.
nftables inet aigw_guard
independent native table. input: drop new container→host. forward: mirror same-plane / cross-plane / ingress / egress / default-drop.
▸ PERMIT
  • reply-direction established traffic
  • same-bridge (same-plane) traffic
  • inbound TCP/443 or lab DNS only when conntrack proves DNAT from the exact published host addr → managed bridge
  • Envoy → 1 resolver (DNS); Envoy → TCP/443 on egress leg
  • Alloy → 1 literal Cribl /32:port (only when enabled)
▸ DROP
  • cross-bridge flows
  • every other bridge-originated physical flow
  • container → host new connections
  • physical src CIDR + dst port alone (never sufficient → IP-forward cannot make host a transit router)
  • any non-Envoy workload reaching the resolver
Docker stays on its iptables backend (experimental nftables backend would drop supported DOCKER-USER integration). Reject target spelled %%REJECT%% in inventory; drift compared on canonical form.

Identity & authorization

Keycloak realm aigw · multivalued roles claim
aigw-users
Open WebUI chat access.
aigw-developers
dev-portal key issuance + tool snippets.
aigw-admins
developer fns + admin-portal + 4 admin-UI edge gates.
ADMIN-UI GATES · 4× oauth2-proxy
Distinct callback + cookie namespace each; enforce aigw-admins. Cookies revalidate every 5 min, expire at 8 h. Grafana runs auth-proxy mode: trusts only fixed oauth2-proxy-grafana src + X-Forwarded-Preferred-Username, auto org Admin, own login + basic auth disabled.
ADMIN MUTATIONS
Require CSRF + fresh step-up prompt=login, max_age=0 within a 5-min window; every page read re-checks live composite roles → revoked admin fails closed with a valid cookie. Isolated anthropic-wif realm for WIF only. Open WebUI = service identity svc-open-webui, not the human.

fixed 172.28.0.2 · sole external identity
litellm · key-rotator → http://envoy-egress:8080/{anthropic,openai}/… over net-vendor → envoy-egress → pinned TLS, exact SNI/SAN, per-vendor trusted_ca → Anthropic / OpenAI
TLS ORIGINATION
No catch-all route. Exact SNI/SAN validation; per-vendor CA bundle narrower than system trust. Startup gate rejects absent / placeholder / system-wide bundles + argv config overrides.
LISTENERS
admin 127.0.0.1:9901 (unauth, loopback). stats 0.0.0.0:9902 → only /stats/prometheus.
FIREWALL EXCEPTION
Only workload allowed external DNS (1 resolver) + TCP/443. No CA-drift monitor — detection via Envoy TLS counters + access logs at/after failure.
