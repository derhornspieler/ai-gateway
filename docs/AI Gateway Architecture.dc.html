03

Layered security enforcement

Seven independent layers, ingress to egress. Each fails closed on its own — compromising one does not disable the others.

L1
Host ingress
firewalld zones
Exact source CIDRs per leg · SSH key-only on ADM · no listener on the egress leg.
L2
Packet policy
DOCKER-USER + aigw_guard
Atomic iptables chain plus an independent native nftables guard deny cross-plane, container-to-host, and unapproved bridge egress.
L3
Network segmentation
18 per-function bridges
Services join only the planes they need · fixed IPs for firewall-addressed workloads.
L4
Identity
Keycloak OIDC
Three realm roles · per-UI oauth2-proxy gates · step-up and live-role rechecks for admin mutations.
L5
Secrets
Vault-backed
Provider credentials in Vault · file-backed Docker secrets · no secret in argv/env · fail-closed blank-variable Compose contract.
L6
Runtime
SELinux + DHI images
SELinux enforcing with per-container MCS · non-root digest-pinned images · read-only binds · no Docker socket exposure.
L7
Egress
Envoy sole identity
Exact routes · exact SANs · per-vendor CA bundles narrower than the system trust store.

05

Developer key lifecycle

Keycloak group membership is the authorization source. LiteLLM virtual keys are always derived from it — and revoked with it.

Developer opens dev-portal
CHECK Token carries aigw-developers?
No → Denied
CHECK Live member of a managed project group? — verified via key-rotator
No → Denied
key-rotator mints scoped LiteLLM virtual key
project ID = group name, stored in key metadata
One-time plaintext key shown once
never stored, logged, or redisplayed — snippets carry only YOUR_KEY
REVOKE

When an admin removes the member from the group, key-rotator logs the user out of Keycloak and deactivates that subject's project keys — both before and after the membership change, closing the mint/removal race.

06

Provider credential rotation — Anthropic WIF

No long-lived vendor key sits in application config. key-rotator brokers a short-lived Anthropic token through Keycloak's isolated anthropic-wif realm using private_key_jwt; the private key exists only in Vault.

key-rotator Vault Keycloak · anthropic-wif Envoy Anthropic API
1key-rotator → Vaultread client private key (ai-gateway/anthropic-wif-client-key)

3Keycloak → key-rotatorworkload identity JWT · aud api.anthropic.com · 600 s lifespan
4key-rotator → EnvoyPOST /v1/oauth/token (WIF exchange)
5Envoy → Anthropicpinned TLS · exact SAN · narrowed CA
6Anthropic → key-rotatorshort-lived sk-ant-oat01 access token
7key-rotatorinstall as LiteLLM provider credential (anthropic-primary) · schedule refresh
