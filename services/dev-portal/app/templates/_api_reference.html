The exact API surface this gateway exposes. Every example uses the YOUR_KEY placeholder — paste the key you copied when you created it. The gateway serves two request shapes over the same virtual key; only the endpoints below are reachable (management and schema paths are blocked at the edge).

Authentication

  • OpenAI shape — header Authorization: Bearer YOUR_KEY
  • Anthropic shape — headers x-api-key: YOUR_KEY and anthropic-version: 2023-06-01
  • 401 = missing/wrong header for the shape, or an inactive key · 403 = a path outside the list below · 429 = your project's TPM/RPM limit.

POST /v1/chat/completions — OpenAI-compatible chat

curl "{{ api_base }}/v1/chat/completions" \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-5",
    "messages": [{"role": "user", "content": "hello"}],
    "max_tokens": 256
  }'

Omit model to use your project's default. Streaming: add "stream": true (server-sent events).

GET /v1/models — list available models

curl "{{ api_base }}/v1/models" -H "Authorization: Bearer YOUR_KEY"

Returns the same list shown above. /v1/models/<id> returns a single record.

POST /v1/messages — Anthropic-compatible messages

curl "{{ api_base }}/v1/messages" \
  -H "x-api-key: YOUR_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-5",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "hello"}]
  }'

This is the endpoint Claude Code and the anthropic SDK use. The gateway injects Anthropic prompt caching on the system message for you.

POST /v1/messages/count_tokens — Anthropic token counting

curl "{{ api_base }}/v1/messages/count_tokens" \
  -H "x-api-key: YOUR_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{"model": "claude-sonnet-4-5", "messages": [{"role": "user", "content": "hello"}]}'

Returns {"input_tokens": N} without running the model.

GET /health/liveliness and GET /health/readiness report gateway health (no auth). /v1/completions and /v1/embeddings are accepted by the edge but no model is configured to serve them, so they are not usable today.
